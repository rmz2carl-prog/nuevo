Estado del árbol de trabajo:
git status

Actualizar repositorio:
git pull

Guardar todo el historial de cambios en un archivo
git log > log.txt

Crear rama nueva:
git checkout -b nombreDeLaRama

cambiar entre ramas:
git checkout nombreDeLaRama

Agregar archivos para confirmar:
git add .

Quitar archivos para confirmar
git reset .
O si solo quieres quitar un archivo específico
git reset nombreDelArchivo.txt
O la extensión que tenga

Confirmar cambios:
git commit -m "Descripción de los cambios"

Subir cambios con rama nueva
git push --set-upstream origin nombreDeLaRama

Subir cambios a rama que ya existe:
git push

Ver las ramas locales:
git branch

Eliminar una rama local:
git branch -D nombreDeLaRama

Cuando se va a empezar a trabajar en algo el orden de los comandos es:
git checkout -b nombreDeLaRama
// haces todo tu trabajo y luego
git status
// acá ves lo que cambiaste o agregaste o quitaste, lo agregas para confirmar
git add .
// aquí agregaste tus archivos para confirmar, revisas con
git status
// Ahora si confirmas tus cambios
git commit -m "Descripción de los cambios"
// ya que está todo listo subes la rama, si es nueva entonces haces set upstream
git push --set-upstream origin nombreDeLaRama
// si ya está subida la rama entonces haces push normal
git push
// para crear el pull request haces
gh pr create
// acá sigues las instrucciones y te da el enlace directo al pull request
// ya que se haga merge del pull request puedes regresar a la rama principal  y borrar la que creaste
git checkout main
git pull
git branch -D nombreDeLaRama
// si da error es porque debes estar en una rama diferente, no puedes borrar ramas en las  que estás parado porque te caes

Si tu pull request tiene conflictos, primero te cambias a tu rama con los conflictos si no estás en ella, y ya en la rama haces:
git rebase origin/main
te dirá que estás en un rebase interactivo, y los archivos con conflictos, revisas con:
git status
entonces modificas los archivos con conflictos para resolverlos
desde <<< HEAD hasta ==== es el código que está en main, y del === hasta >>> es lo que tu cambiaste
decides cual es el código bueno y solo dejas ese
asegurate de eliminar los <<< HEAD, los === y los >>>> junto con los números y letras que tiene como al azar
ya que corregiste el archivo entonces haces:
git add .
revisas que todo esté verde:
git status
y le dices que continue:
git rebase --continue
si hay más conflictos te volverá a decir
de nuevo corriges ese código, agregas con add . y continuas con --continue hasta que te diga success
revisas por última vez que todo esté limpio:
git status
y subes la corrección de los conflictos con:
git push -f
los cambios aparecen luego luego en el pull request y ya te debe dejar hacer merge
